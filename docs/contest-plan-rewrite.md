# T-Sync 공모전 기획서 개정안

## 1. 개발 결과물 개요

### 서비스명
T-Sync

### 한 줄 소개
실시간 버스 위치와 정류장 순서 데이터를 기반으로 환승 성공 가능성을 예측하고, 실패 위험이 높을 때 더 안전한 대안 환승 전략을 추천하는 웹 기반 대중교통 의사결정 지원 서비스

### 서비스 요약
T-Sync는 전국 통합개방데이터 중 `전국 초정밀 버스 실시간 위치 정보`를 핵심 데이터로 활용하여, 사용자가 실제 환승에 성공할 수 있는지를 사전에 판단해 주는 서비스이다. 기존 대중교통 앱이 단순 ETA 중심 정보를 제공하는 데 비해, 본 서비스는 현재 탑승 중인 차량의 위치, 목표 노선 차량의 접근 시점, 환승 정류장 기준 보행 시간을 함께 고려해 환승 성공 확률을 제시한다. 또한 환승 실패 가능성이 높을 경우 같은 노선의 다음 차량 대기, 다음 공통 정류장 환승 등 실제 실행 가능한 대안 전략을 함께 추천한다.

## 2. 개발 결과물의 기능 및 특징

### 개발 배경
현재 대부분의 대중교통 서비스는 차량 도착 예정 시간이나 최단 경로 중심으로 안내한다. 그러나 실제 환승 상황에서는 첫 번째 차량 지연, 두 번째 차량 조기 도착, 정류장 내 이동 시간 부족 등으로 인해 안내된 경로대로 이동하더라도 환승에 실패하는 경우가 많다. 특히 통근자, 학생, 고령자처럼 시간 여유가 적거나 이동 속도가 다른 사용자에게는 단순 ETA보다 환승 성공 가능성 자체가 더 중요한 정보가 된다.

### 핵심 기능
1. 실시간 차량 위치 기반 환승 성공 확률 예측
   현재 탑승 차량의 위치와 목표 노선 차량의 위치를 비교해 환승 정류장 기준 도착 시점을 계산하고, 환승 성공 확률을 수치로 제공한다.
2. 사용자 이동 프로필 기반 환승 시간 반영
   빠른 이동형, 일반 이동형, 느린 이동형으로 나누어 환승 여유 시간을 다르게 계산함으로써 사용자 상황을 반영한다.
3. 환승 실패 위험 경고
   환승 여유 시간이 부족한 경우 안정, 불확실, 위험의 세 단계로 위험 수준을 구분해 직관적으로 안내한다.
4. 대안 환승 전략 추천
   기본 환승이 어렵다고 판단되면 같은 노선 다음 차량 대기 또는 다음 공통 정류장 환승 등 성공 가능성이 더 높은 전략을 추천한다.
5. 웹 데모 기반 시연
   사용자가 노선과 환승 정류장을 선택하면 입력, 처리, 출력 흐름을 한 번에 확인할 수 있는 데모 형태로 구현한다.

### 시스템 구성
1. 데이터 수집 계층
   공공데이터 API를 통해 노선 정보, 정류장 정보, 실시간 차량 위치를 수집한다.
2. 데이터 저장 계층
   PostgreSQL에 노선, 정류장, 노선-정류장 매핑, 차량 위치, 예측 결과를 저장한다.
3. 분석 및 예측 계층
   실시간 위치와 정류장 순서, 보행 버퍼를 조합해 환승 성공 확률과 추천 전략을 계산한다.
4. 서비스 제공 계층
   FastAPI 기반 백엔드와 웹 프론트엔드를 통해 결과를 사용자에게 제공한다.

## 3. 기존 서비스와 차별성 및 독창성

### 기존 유사 서비스와의 차별점
기존 버스 안내 서비스는 대부분 “언제 도착하는가”를 알려주지만, T-Sync는 “지금 환승이 실제로 가능한가”를 판단한다. 이는 정보 제공형 서비스에서 의사결정 지원형 서비스로의 확장이라는 점에서 차별성이 있다.

### 독창성
1. 실시간 버스 위치 데이터와 정류장 순서를 결합해 환승 상황만을 별도로 분석한다.
2. 사용자 보행 특성을 반영해 같은 경로라도 사람마다 다른 환승 성공 확률을 계산한다.
3. 실패 위험을 단순 경고로 끝내지 않고, 성공 가능성이 더 높은 대안 전략까지 제시한다.
4. 공공데이터를 단순 조회형이 아닌 예측형 서비스로 전환한다.

### 심사 포인트 대응
1. 참신성
   기존 ETA 중심 서비스와 다르게 환승 실패를 예측하고 대안을 추천하는 구조를 갖는다.
2. 국민편익
   통근 시간 손실 감소, 대중교통 이용 스트레스 완화, 교통약자 배려형 안내로 이어질 수 있다.

## 4. 공공데이터 등 데이터 활용성

### 필수 활용 데이터
전국 통합개방데이터 7종 중 `전국 초정밀 버스 실시간 위치 정보`를 필수 활용하였다.

### 활용 데이터와 역할
1. 노선 기본 정보 데이터
   노선 식별, 노선 번호 제공, 환승 후보 노선 확인에 활용
2. 노선 경유지 정보 데이터
   각 노선의 정류장 순서와 공통 정류장 탐색에 활용
3. 실시간 차량 위치 데이터
   현재 차량 접근 상황, 목표 노선 차량 도착 시점 계산에 활용

### 데이터 획득 및 가공 방식
공공데이터 API를 통해 노선, 정류장, 차량 위치 데이터를 수집하고, PostgreSQL에 적재한 뒤 노선별 정류장 순서와 실시간 위치를 결합하여 환승 예측에 필요한 형태로 가공하였다.

### 데이터 활용성 강조 포인트
1. 단일 데이터 조회가 아니라 노선 정보, 정류장 정보, 차량 위치를 결합하여 실제 서비스 로직으로 전환하였다.
2. 실시간 데이터가 예측 기능과 추천 기능의 핵심 입력으로 사용된다.
3. 향후 교통약자 이동지원, 공영자전거, 실시간 신호등 데이터 등과 연계해 서비스 확장이 가능하다.

## 5. 개발 결과물의 기술성과 구현 및 실현가능성

### 주요 기술
1. FastAPI 기반 REST API 서버
2. PostgreSQL 기반 데이터 저장 및 조회 구조
3. Docker Compose 기반 실행 환경 구성
4. 실시간 차량 위치 기반 환승 확률 계산 로직
5. 대안 환승 전략 추천 로직

### 실제 구현 범위
1. 노선, 정류장, 차량 위치 데이터 수집 및 저장
2. 노선별 정류장 조회 API 제공
3. 실시간 차량 위치 조회 API 제공
4. 환승 성공 확률 계산 API 제공
5. 위험도 판단 및 추천 전략 반환 API 제공

### 구현 가능성
본 결과물은 공공데이터를 실제로 수집하여 데이터베이스에 적재하고, 백엔드 API를 통해 예측 결과를 반환하는 데모 버전으로 구현하였다. 심사위원이 입력, 처리, 출력 흐름을 바로 확인할 수 있도록 웹 기반 데모 화면을 제공하며, 향후 프론트엔드 고도화 및 모델 정교화가 가능한 구조로 설계하였다.

### AI 활용 계획 및 가점 대응
현재 데모 버전은 실시간 위치, 정류장 순서, 보행 버퍼를 활용한 예측 로직을 우선 구현하였다. 향후 실제 운행 이력 데이터를 누적하면 Random Forest 등 지도학습 모델을 적용해 확률 예측 정밀도를 높일 수 있으며, 대안 경로 선택에도 AI 기반 랭킹 모델을 적용할 계획이다. 본 서비스는 공공데이터 기반 예측과 추천 기능을 포함하므로 AI 활용 가점 항목에도 부합한다.

## 6. 기대 효과

### 국민 생활 개선 효과
1. 환승 실패 감소로 통근, 통학 시간 낭비를 줄일 수 있다.
2. 사용자는 실패 가능성이 높은 환승을 미리 피할 수 있어 이동 스트레스를 줄일 수 있다.
3. 이동 속도가 다른 사용자에게 맞춤형 판단 기준을 제공할 수 있다.

### 파급 효과
1. 공공데이터를 조회형 서비스에서 예측형 서비스로 확장한 활용 사례가 될 수 있다.
2. 전국 단위 버스 데이터 기반으로 지역 확장성이 높다.
3. 향후 교통약자 이동지원, 공영자전거, 신호등 정보 등과 연계하면 통합 교통 의사결정 플랫폼으로 발전 가능하다.

### 지속가능성
1. 현재 구조는 지역별 노선 데이터만 확보되면 확장이 가능하다.
2. 실시간 데이터 적재와 예측 API가 분리되어 있어 기능 고도화에 유리하다.
3. 향후 예측 정확도 개선, 사용자 피드백 반영, 앱 전환 등 서비스 확장 여지가 충분하다.
